Comment 2 by vbarathan, Feb 10, 2009 
Yes this is an issue in the sample.  Thanks for reporting.

This seems to be a spreadsheets api server issue.  We are investigating this
(internal bug tracker id: 1641977).   You can explicitly set the service version to
v1 using the following statement:
this.service.setProtocolVersion(SpreadsheetService.Versions.V1);

unfortunately there is another data model bug which is blocking the above workaround
as well.  This issue will be fixed in the next java client release (expected this week).

Summary: with this week's release of java client lib, you should be able to use the
above workaround to do batch operations.

For one, your image variable is out of scope in your paint method. You declared it in the init method, so it doesn't exist to paint.


Also my guess is that Java cannot find your image. When you specify getCodeBase() it means that java should search in the directory with the .class file in a folder called C:\dp.jpg

So java is probably looking for something like:

file:/C:/Documents%20and%20Settings/James/workspace/GUI/bin/C:/dp.jpg

Put your dp.jpg file in the folder with your .class and have:

code:

getImage(getCodeBase(),"dp.jpg");

Yeah that should work

Not sure what's causing the null pointer.

It might have to do with you using Ready To Program (it's not quite Java imo).

This is the code I used:

code:

//The "Final_Game" class.
import java.applet.*;
import java.awt.*;
import java.awt.Image;


public class Final_Game extends Applet
{
// Place instance variables here
Image myImage;

public void init ()
{
myImage = getImage(getCodeBase(), "dp.jpg");
System.out.println(getCodeBase());
// Place the body of the initialization method here
}


public void paint (Graphics g)
{
// Place the body of the drawing method here
g.drawImage(myImage, 300, 200, 60, 120, this);
} // paint method
} // Final_Game class

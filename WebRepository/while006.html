Install the latest version of java and configure the JAVA_HOME PATH CLASSPATH properly, so that it will use the proper jvm.
Please some configuration tips here: http://vietpad.sourceforge.net/javaonlinux.html

And now try to start the eclipse again. (Its better if U get the ecliplse** extractor from http://www.eclipse.org/downloads )
If you still find the issue, read on.

In some cases, on an x86 (32 bit) system running RHEL 5.2, Eclipse (from eclipse.org) will fail to start with:

/usr/bin/java: symbol lookup error:
/usr/lib/eclipse/configuration/org.eclipse.osgi/bundles/9/1/.cp/libswt-mozilla-gtk-3235.so:
undefined symbol: _ZN4nsID5ParseEPKc


To work around this problem, the xulrunner package must be removed:

# yum remove xulrunner

-- Now start the ecliplse, it should work (as is did for me)

As firefox will also be removed in the previous (xulrunner) step, you can reinstall it:
# yum install firefox

The error happen because of class path setting
just try to make a link to the the default path as follows
inorder to link java installed at a separate location to the default path i used
root@micky/~/autosel/jdk1.6.0_07/bin#ln -s /root/autosel/jdk1.6.0_07/bin/java /usr/bin/java
root@micky/~/autosel/jdk1.6.0_07/bin#ln -s /root/autosel/jdk1.6.0_07/bin/javac /usr/bin/javac
thi s will link java and javac which is in root/autosel/jdk1.6.0_7/bin to the default /usr/bin/java and javac
so as i done just use ln -s to link the java file to usr/bin/java or wat ever .
this will fix the issue

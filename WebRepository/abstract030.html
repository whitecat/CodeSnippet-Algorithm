> Is there anything called "abstract interface" in java. if so what
> are its uses ??

When you're defining an interface in Java, it's always abstract, though you
don't need to declare them as such.

So there's no difference between

abstract interface I
{
void method();
}

....and...

interface I
{
void method();
}

For the use of Java interfaces in general you can look at:

http://mindprod.com/jgloss/interfacevsabstract.html
http://mindprod.com/jgloss/interface.html

Yeah, what Bjorn said. A Java Interface is basically an abstract,
virtualized object.

They're better than objects (according to Java fans) because they don't
introduce the same complexities that multiple inheritance does.

In other words, an object can extend (inherit from) another object, and
also implement one or more interfaces, and the compiler and run-time
have a much easier job than if Java allowed you to extend multiple objects.

So there's on of its uses right there: Interfaces are how one does
multiple inheritance in Java.

The key words you need to know about are Interface, Implements, and for
regular objects Extends. Good luck!

Jabstract: A full Abstract Syntax and Parser for Java

Sebastian Danicic

Abstract:

A System for aiding Tranformation and Analysis of Java Programs

Abstract Syntax Trees

See Source code
and Javadoc Documentation
Source Code Tar Ball(classes.tar.gz)

Introduction

My research interests include program transformation (for example Program Slicing and Testing) and analysis. What my group requires is a system which enables us to perform program transformation and analysis of Java Programs. In order to do this the following things are required:
  • A full Abstract Syntax for Java.
  • A parser which converts Java programs into this Abstract Syntax.
  • Tools for manipulating the Abstract Syntax Tree
  • An unparser (or pretty printer) for converting the transformed AST back to Java source code.

The System

So far I have produced a system for Java 1.1. The system was implemented using a tool from Sun Microsystems: JavaCC, a parser generator a preprocessor for JavaCC [tm] that inserts parse tree building actions at various places in the JavaCC source. This tool comes with some ready made Java grammars which help enormously in the production of our System. The one we used was Java1.1.jj. This is the grammar that includes the Java 1.1 language extensions. It is based on the Java Language Specification by Gosling, Joy, and Steele augmented by the inner class document released with JDK 1.1.

Using the grammar Java1.1.jj an Abstract Syntax was defined. This consists of over fifty classes, one for each Syntactic Category for Java 1.1. Since the classes reflect the Syntax of Java 1.1 rather than the general purpose concrete syntax trees produced by JJTree they make the analysis, manipulation and transformation of Java Programs much easier.

One interesting feature of our AST is that program comments are included in the Abstract Syntax.

This means that when transforming programs for example in slicing, imay be possible that relevant comments may be kept while irrelevant comments will be removed.

As pretty printer for Java 1.1 has been written in terms of AST. Initial testing indicates success. The pretty printer has the pleasing property that comments are retained.

The Pretty Printer

You can use the pretty printer to test the system. To test:
  • Untar the Source Code Tar Ball(classes.tar.gz)
  • Add the new directory created by untarring to your CLASSPATH
  • Try soemthing like java JavaTools.PrettyPrintWithComments test.java 0 where test.java is a Java program that you want to pretty print. You will end up with an extra file called TEMP which is the file without any comments.
